# 부팅 자동시작 테스트 매트릭스 (v1.3.17~)

부팅 자동시작 / 백그라운드 시작 동작은 v1.3.4부터 환경에 따라 간헐적으로
재발해 왔습니다. 매 빌드 제출 전, 아래 시나리오를 **진료실 PC(MSIX 설치본)**
에서 직접 확인하세요. 한 항목이라도 ❌이면 제출하지 않습니다.

## 사전 준비
- `scripts/preflight/verify-msix.ps1` 통과 (제출 전 사전 점검)
- 사이드로딩본을 진료실 PC에 설치 (`scripts/sideload/install.ps1`)
- `scripts/preflight/post-install-check.ps1` 통과

## 시나리오

| # | 사용자 액션 | 기대 동작 | 확인 방법 |
|---|------------|-----------|-----------|
| 1 | 신규 설치 후 시작 메뉴에서 수동 실행 | 메인 창 정상 표시, 크래시 없음 | 창이 보이는지 육안 확인 |
| 2 | 자동 실행 **ON** + 백그라운드 **OFF** → 재부팅 | 부팅 후 메인 창 표시 | 로그온 후 창 표시 확인 |
| 3 | 자동 실행 **ON** + 백그라운드 **ON** → 재부팅 | **트레이만**, 메인 창 안 보임 (깜빡임 없음) | 트레이 아이콘만 있고 창 없음 |
| 4 | 자동 실행 **OFF** → 재부팅 | 앱 실행 안 됨 | 트레이/창 모두 없음 |
| 5 | Windows 설정 > 시작프로그램에서 OFF → 앱 토글 시도 | "사용자가 Windows 설정에서 비활성화함" 안내, 토글 비활성 | 설정 다이얼로그 확인 |
| 6 | 자동 실행 ON → 앱에서 토글 OFF → 다음 부팅 | 앱 실행 안 됨 | 부팅 후 미실행 확인 |
| 7 | 백그라운드(ON)로 부팅 → 트레이 아이콘 클릭 | 메인 창 정상 표시 | 트레이 클릭 → 창 복원 |
| 8 | 백그라운드 시작 후 녹음 시작/중지 | 녹음 정상 동작, 파일 생성 | 저장 폴더에 파일 확인 |
| 9 | 부팅(백그라운드) → 설정 > 진단 패널 열기 | StartupTask=enabled, 부팅 백그라운드=ON, 부팅 이력에 "트레이로 시작" 기록 | 진단 패널 항목 확인 |
| 10 | 부팅 → 로그 폴더 확인 | logs/ 안에 오늘 날짜 `eyebottle_YYYYMMDD.log` 존재 | 진단 패널 "로그 폴더 열기" |

## 핵심 회귀 포인트 (v1.3.17 구조 변경)
- **시나리오 3**이 이 릴리스의 핵심. native(main.cpp)가 `--autostart`일 때
  `Show()`를 호출하지 않으므로 "창 깜빡임"조차 없어야 정상.
- **시나리오 2**: 자동시작이지만 백그라운드 OFF → Dart가 명시적으로 `show()`를
  불러야 창이 뜸. native가 숨겨둔 상태이므로 이 경로가 깨지면 "자동시작인데
  창이 안 뜨는" 새 회귀가 됨. 반드시 확인.
- **시나리오 9~10**: 로그/진단이 남는지 확인 — 다음에 또 문제가 생겨도
  `diagnose.ps1` 한 번으로 원인 캡처 가능해야 함.
